The `prpc` CLI is your companion for building and debugging pRPC services. It is provided by the `prpc-codegen` package.

## Commands

### `serve`
Start an ASGI server for any module containing a pRPC application.

```bash
prpc serve my_app.main --port 8000 --reload
```

- `--host / -h`: Bind to a specific address (default: `127.0.0.1`).
- `--port / -p`: Port to listen on (default: `8000`).
- `--reload`: Enable auto-reload on code changes.

### `inspect`
Visualize all registered RPC procedures in a module.

```bash
prpc inspect my_app.main
```

This prints a beautiful table showing method names, parameters, return types, and docstrings.

### `codegen`
Generate a client for a specific target language.

```bash
prpc codegen -m app.main -t ts -o client.ts
```

- `-m / --module`: The Python module to introspect.
- `-t / --target`: Target language (currently only `ts` is supported).
- `-o / --output`: Output file path (default: `client.ts`).

### `version`
Show the current pRPC version.

```bash
prpc version
```
